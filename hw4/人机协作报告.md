### promopt
```python
import pandas as pd
import numpy as np

# 假设 df 是你已经加载好的数据集
print(f"清洗前总行数: {len(df)}")

# --- 1. 基础逻辑清洗 ---
# 计算行程耗时（秒）
df['trip_duration'] = (df['tpep_dropoff_datetime'] - df['tpep_pickup_datetime']).dt.total_seconds()

# 规则A：剔除下车时间早于上车时间（耗时<=0）的记录
df = df[df['trip_duration'] > 0]

# 规则B：剔除行程距离、基础车费非正数的记录
df = df[(df['trip_distance'] > 0) & (df['fare_amount'] > 0)]

# 规则C：剔除乘客数量异常（0人或超过6人）的记录
# 注意：passenger_count是float类型，这里直接做数值比较
df = df[(df['passenger_count'] >= 1) & (df['passenger_count'] <= 6)]

# --- 2. 空间地理围栏清洗 ---
# 规则D：剔除超出纽约市大致边界的上下车坐标
# 经度范围：-74.03 到 -73.75，纬度范围：40.63 到 40.85
lon_min, lon_max = -74.03, -73.75
lat_min, lat_max = 40.63, 40.85

df = df[
    (df['PULocationID'] > 0) &  # 确保区域ID有效（如果直接用经纬度列可替换为经纬度判断）
    (df['DOLocationID'] > 0) &
    # 如果你的数据里有具体的经纬度列（pickup_longitude等），可以用下面这四行替换上面的ID判断：
    # (df['pickup_longitude'].between(lon_min, lon_max)) &
    # (df['pickup_latitude'].between(lat_min, lat_max)) &
    # (df['dropoff_longitude'].between(lon_min, lon_max)) &
    # (df['dropoff_latitude'].between(lat_min, lat_max))
    (df['trip_duration'] <= 3600 * 6) # 顺便剔除耗时超过6小时的极端长途单
]

# --- 3. 统计离群值清洗 (以基础车费 fare_amount 为例) ---
# 规则E：使用IQR法剔除极端车费
Q1 = df['fare_amount'].quantile(0.25)
Q3 = df['fare_amount'].quantile(0.75)
IQR = Q3 - Q1
lower_bound = Q1 - 1.5 * IQR
upper_bound = Q3 + 1.5 * IQR

# 也可以直接简单粗暴地限制一个最大合理值，比如200美元
df = df[(df['fare_amount'] >= lower_bound) & (df['fare_amount'] <= 200)]

print(f"清洗后总行数: {len(df)}")
print(f"共剔除异常数据: {len(df) - len(df)} 行")

```

### 1. AI 协作带来的效率飞跃
在本次项目开发中，AI 扮演了“全能技术顾问”的角色，极大地提升了我的开发效率。
- **打破知识盲区**：在面对不熟悉的 Python `cmd` 库构建问答系统时，AI 不仅提供了基础代码，还在我遇到 `Unknown syntax` 报错时，迅速定位到 `default` 函数缺失的问题。这种即时的反馈机制，比传统查阅文档要高效得多。
- **工程化思维的建立**：在 M3 阶段，我最初只是简单地运行训练脚本，导致每次都要重复训练。AI 主动提出了“模型持久化（保存为 .h5 文件）”和“构建 main.py 一键入口”的工程化建议。这让我跳出了“写脚本”的思维，开始用“开发软件”的标准来要求自己。

### 2. 遇到的挑战与局限
虽然 AI 非常强大，但在协作过程中我也发现了一些需要注意的地方：
- **上下文与环境的适配**：AI 生成的代码有时是通用的，比如在处理文件路径时，它最初可能使用绝对路径。我需要人工介入，将其修改为 `data/` 和 `outputs/` 这样的相对路径，以确保代码在不同电脑上都能运行。
- **业务逻辑的主导权在我**：AI 可以写出完美的代码框架，但它不知道我想分析“纽约出租车”的数据。例如在 M4 中，关于“什么是高峰时段”、“如何定义热门区域”，这些核心业务逻辑必须由我通过提示词明确告诉 AI，它才能生成有价值的代码。

### 3. 总结与成长
通过这次人机协作完成大作业，我深刻认识到：**AI 是最好的副驾驶，但驾驶员必须是我自己。**
我学会了如何精准地向 AI 提问（Prompt Engineering），如何拆解复杂需求（如将大作业拆分为 M1-M4），以及如何审查和调试 AI 生成的代码。这种“人类负责逻辑与审核，AI 负责实现与补全”的协作模式，将是我未来学习和工作中非常重要的能力。
